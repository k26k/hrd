추상 클래스 (기울임체로 표기)
- abstract 예약어를 사용한다.
- 실체 클래스는 객체를 직접 생성할 수 있는 클래스이고 
  이런 클래스들의 공통적인 특성만을 추출해서 선언한 클래스가 추상 클래스이다.
- 실제 클래스를 만들때 사전에 정의된 필드 명과 메서드 명을 상속받아서
  실체 클래스들의 필드와 메서드의 이름을 통일할 목적으로 사용한다.
- 추상 클래스는 객체 생성이 불가능
  추상 클래스를 상속받은 자식 클래스로 생성해야한다.

추상 메서드 (기울임체로 표기하고 구현부가 없음)
- 추상 메서드도 abstract 예약어를 사용한다.
- 메서드를 구현하지 않고 선언만 한다. { } 구현부가 없다.
- 상속받는 실체 클래스는 추상메서드를 필수적으로 구현해야 한다.


인터페이스
- 모든 메서드가 추상메서드(abstract method)로 이루어진 클래스이다.
- 역할 : 클래스 혹은 프로그램이 제공하는 기능을 명시적으로 선언하는 역할
  인터페이스만 봐도 어떤 매개변수가 사용되는지 또는 어떤 자료형이 반환되는지 알 수 있다.

인터페이스로 구현 객체를 사용하는 방법
1. 필드
2. 생성자의 매개변수
3. 메서드의 지역변수
4. 메서드의 매개변수
